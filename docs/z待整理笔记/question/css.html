1、制作网页的流程？

一直都在想这个问题，谢了这么久的代码，却对这个问题的答案模糊不定。
要回答这个问题，我觉的离不开三样东西，html、css、js、兼容。
一、明白网页要需要在那些环境下运行。
知道自己写代码的准则还是很重要。需要兼容哪些浏览器和设备？


一、写网页需要先研究设计图。
研究设计图，主要是看整个网页的大结构和找出一些可重复利用的样式。
所谓的结构就是：
从顶层的元素开始分析，从粗糙到精细。一般只关注盒子的大小和布局。如果元素有边框
则外边距也要考虑，否则要考虑内嵌元素外边距和并的问题。

可重复的样式：
如果你想找到一种很复杂样式表可以完全应用到好几个不同的元素上这几乎是不可能的。
所以一般都是先满足大众化 的元素然后在具体满足个别的元素。
写css有个很好地办法，就是先满足大众，然后满足个别。


二、设计好html结构。
一个好的结构能省去很多麻烦。
在尽可能的情况下使用语义化标签。


三、布局和样式。
布局：
逐级布局。以前总是喜欢，按照网页从上到下的顺序去写，这样有时并不好。
在布局的过程中边框以内的东西先不管。先把架子搭起来。
只考虑，尺寸，边框，布局，外边距。

样式：
写样式的顺序：
（1）width、height，border
（2）布局
（3）内部（font、bg、）
最好遵从从外到内的顺序。


四、js互动。


五、兼容。
一般都是写一点测一点，有层次的。从布局到样式。

1、非法嵌套会导致定位失效，比如在p内的ul不会随着p的定位而移动。 标签要合理嵌套。

2、用css写三角形

    width:0;
    height:0;
    overflow:hidden;
    border-left:100px solid transparent;
    border-right:100px solid transparent;
    border-bottom:100px solid #ca0309;

3、注意选择器嵌套过多的问题

有时候因为很html结构前套过多，导致选择器嵌套过多，一般不要超过3个。 应该想办法避免。

选择器并用

选择器.x1.x2匹配哪些同时有.x1、 .x2类的元素。

遮住屏幕

        position:fixed;
        top:0;
        right:0;
        bottom:0;
        left:0;

每一个有宽高的元素加了overflow:auto;都可看做是一个视口。

button激活时chrome里有边框 

        button:focus{
            outline:0;
        }

两个表单元素input换行被解析

z-index

两个定位兄弟元素，如果想让其中一个兄弟元素内部的元素在另一个 兄弟元素的上面，Ie7必须在父级元素上加才有效果。
