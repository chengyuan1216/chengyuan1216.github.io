什么是webpack？

它是一个前端工具，可以让各个模块进行加载、预处理，再进行打包。
其优点如下：
1、支持commonJS和AMD模块。
2、支持很多加载器的调用，比如babel-loader加载器能使我们使用es6的语法来编写代码。

我个人的理解就是能够让我们使用浏览器不支持的语法写代码，最后将所有的模块打包成为 浏览器能支持的代码。

如何安装配置webpack？

1、webpack 
         -main.html
            -src
             -react.min.js
             -main.js
2、npm init
在webpack文件夹内生成package.json文件。
3、npm install -g webpack
全局安装webpack。
安装好了后会生成node_modules文件夹，里面内包含了之前安装的webpack。

如果要安装在项目下，则：
npm install webpack --save-dev
4、配置webpack
每个目录下都必须有一个webpack.config.js文件。这是一个项目的配置，就是告诉webpack需要做什么。
下面是一个常见的配置： 
            moudle.exports = {
                //页面的入口文件，比如这里的main.js
                entry:"./src/main.js",
                //指定打包后生成的目标文件名，及路径
                output:{
                    filename:"build/build.js"
                }
                moudle:{
                    loaders:[
                        //css文件使用style-loader 和 css-loader来处理
                        //test是正则表达式，对符合的文件名使用响应的加载器。
                        { test:/\.css$/ , loader:"style!css" },
                        //.js文件使用jsx-loader来处理
                        { test:/\.js$/ , loader:"jsx-loader" }
                    ]
                },
                //定义了解析模块路径时的配置
                //常用的是extensions用来指定模块的后缀。
                //这样在引入模块时就不需要写后缀，会自动补全
                resolve:{
                    extensions:['', '.js', '.jsx']
                },
                //插件
                plugins:[]
            }

5、安装加载器

npm install jsx-loader --save-dev
npm install style-loader --save-dev
npm install css-loader --save-dev

6、运行

进入到项目文件夹内执行webpack命令。
也可以使用 webpack --display-error-detail,可以查看详细的信息。

webpack下的几个命令

webpack
最基本的启动。
webpack -w
提供watch方法实时进行打包更新
webpack -p
对打包后的文件压缩。
webpack -d
提供source map方便调试代码
